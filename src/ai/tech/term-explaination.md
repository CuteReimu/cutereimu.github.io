在人工智能领域，尤其是生成式AI（如大语言模型）和应用开发平台中，有这样一些重要的术语：

[[toc]]

## Prompt（提示词 / 提示语）

Prompt 是用户输入给 AI 模型（如 ChatGPT、Midjourney 等）的**指令、问题或文本**，目的是引导模型生成特定的输出。它是人与模型交互的**接口**。Prompt 的质量直接决定了 AI 输出的质量。这也就是为什么现在有 “Prompt Engineering”（提示工程）这个职业——通过设计和优化输入文本来引导模型生成更准确、更有价值的结果。

一个结构良好的 Prompt 通常包含以下要素：
- **指令：** 明确告诉 AI 要做什么。例如：“翻译这段文字”、“写一封邮件”、“总结这篇文章”。
- **上下文/背景：** 给 AI 提供额外的信息，帮助它更好地理解。例如：“假设你是一名资深的营销专家”、“基于以下会议记录”。
- **输入数据：** 用户希望 AI 处理的具体内容。例如：一段需要翻译的外语、一份需要总结的报告。
- **输出格式：** 指定 AI 以什么形式回复。例如：“用表格形式输出”、“返回 JSON 格式”、“分三点列出”。

::: note 示例

- 简单 Prompt：
  > 什么是人工智能？
- 复杂 Prompt（含有上下文和格式要求）：
  > **指令**：请用通俗易懂的语言向小学生解释“引力”这个概念。\
  > **要求**：包含一个生活中的例子，并且字数不超过 150 字。\
  > **格式**：请用一段话完成。

:::

## Skill（技能 / 插件 / 能力）

Skill 在不同的 AI 平台有不同的叫法（如 OpenAI 的 GPTs 的 Actions、字节跳动的 Coze 平台称之为插件或技能，或早期智能音箱的 Skills），但其核心含义是：**AI 模型为了完成特定任务而调用的一种外部工具、API 接口或预设的功能模块**。它扩展了 AI 大模型本身的能力，让 AI 不仅能“说”，还能“做”。

大语言模型本身是“静态”的，它们只有训练时学到的知识，无法获取实时信息或操作外部世界。Skill 打破了这种限制。

::: tip 常见的 Skill 类型

-   **知识检索：** 联网搜索（如让 AI 查询今天的实时新闻）。
-   **代码解释器：** 让 AI 写代码并运行，进行数据分析、图表绘制、文件格式转换等。
-   **图片理解/生成：** 调用 DALL-E 或 Stable Diffusion 等模型进行绘图，或调用图像识别模型分析图片内容。
-   **自定义 API 调用：** 连接外部系统。例如，一个“查快递”的 Skill 会调用快递公司的 API；一个“订酒店”的 Skill 会调用订房平台的 API。
-   **预设工作流：** 在某些平台上，Skill 也可能指一个封装好的、包含多个步骤的复杂提示词或自动化流程。

:::

::: note 示例

假设你有一个“旅行规划助手”AI，你问：“帮我规划一下下周末去北京的行程，并告诉我那时的天气。”

AI 为了回答这个问题，可能需要调用两个 **Skill**：
1. **调用【联网搜索 Skill】**：去网上搜索“北京 下周末 天气”。
2. **调用【地图/景点查询 Skill】**：查询北京热门景点、开放时间和交通路线。

最后，AI 综合这些 Skill 返回的数据，结合自己的语言能力，生成一份包含天气和行程的回复给你。

:::

在现代 AI 应用开发中，Prompt 和 Skill 往往一起工作：

1.  **Prompt 负责“理解”和“生成”**：用户通过 Prompt 告诉 AI 自己想要什么。
2.  **Skill 负责“执行”和“获取”**：AI 理解 Prompt 后，判断需要调用哪些 Skill 来完成这个任务（例如查数据、算结果、画图片）。
3.  **结果整合**：AI 将 Skill 执行的结果带回对话中，结合自己的语言能力，生成最终的回复。

**总结一下：**

-   **Prompt** 是你和 AI 沟通的**语言**。
-   **Skill** 是 AI 完成任务的**工具**。

掌握如何写好 Prompt，并了解 AI 具备哪些 Skill，是高效使用当前 AI 工具的关键。

## Buff

**Buff** 是一个非常形象化的术语。如果你熟悉游戏，就很容易理解——在游戏中，给角色加一个“Buff”意味着让他变得更强大、跑得更快或防御更高。

在AI语境下，**Buff 指的是在 AI 模型处理你的请求之前，系统自动对输入内容（Prompt）进行的一种“增强”、“修饰”或“预处理”**。

**Buff** 不是一个独立的组件（像 Skill 那样），而是一个**中间处理层**。它的作用是：

1.  **接收你输入的原始 Prompt**。
2.  **按照预设的规则，自动添加额外的信息、指令或格式**。
3.  **将增强后的“新 Prompt”发送给大模型**。

这样做的目的是**在不改变用户使用习惯的前提下，显著提升模型输出的质量**。

::: note 常见的 Buff 类型与例子

**角色设定 Buff**

你输入了一句话，系统自动在前面加上一句角色扮演指令。

- 你的输入：
  > 帮我总结一下这份会议记录。
- Buff 加持后的输入：
  > 你是一名拥有10年经验的高级项目经理，擅长从混乱的信息中提炼重点。请用专业、简洁的语言，帮我总结一下这份会议记录：[附上你的会议记录]

---

**安全与护栏 Buff**

为了防止模型说出不当言论，系统会在后台自动加上安全约束。

- 你的输入：
  > 如何说服别人相信我？
- Buff 加持后的输入：
  > 请回答：“如何说服别人相信我？”。但需确保内容积极健康，不涉及欺诈、PUA或操纵他人的技巧。

:::

**为什么要用 Buff？**
- **降低用户使用门槛**：用户不需要成为“提示词工程师”（Prompt Engineer），不需要每次都输入复杂的指令。只需要简单说一句，系统自动帮他加料。
- **保证输出一致性**：在企业应用中，通过 Buff 可以强制所有员工使用的 AI 都遵循公司的品牌语调、安全规范和数据格式，不会因为员工个人表达习惯不同而导致 AI 输出跑偏。
- **实现“无感”增强**：用户可能根本没意识到，自己简单的一句话，经过 Buff 层的包装，已经变成了一个包含上下文、角色设定、知识库引用的超级 Prompt。

Buff 就像是 AI 应用的“增效器”。它在用户和模型之间默默地工作，把你的 Prompt 打磨成更精细、更易被模型理解的内容，从而得到更好的结果。

## 总结

我们举一个完整流程的例子，来作为本文的收尾：

1.  你问：**“帮我写一封拒绝客户的邮件。”（这是 Prompt）**
2.  系统自动加 **Buff**：系统心想：“用户没说语气，我默认加一个‘礼貌且委婉’的Buff。”此时发往大模型的实际内容变成了：“请用**极其礼貌、委婉且感谢对方兴趣**的语气，帮我写一封拒绝客户的邮件。”
3.  模型觉得可能需要参考之前的沟通记录，于是调用 **Skill**：去邮箱或CRM系统里查找该客户的往来邮件。
4.  模型结合 Buff 的要求和 Skill 查到的数据，生成最终的邮件回复给你。
